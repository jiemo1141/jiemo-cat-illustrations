# QA Checklist（猫版）

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有小黑猫。
- 小黑猫承担核心动作，不只是装饰。
- 小黑猫是圆润团子身体、黄色大眼睛、粉色鼻子、有腮红。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。

## 失败信号

出现以下情况，重生成或局部编辑：

- 小黑猫被画成别的动物（狗/兔子/普通黑猫）。
- 眼睛不是黄色。
- 小黑猫太拟人化（穿衣服戴帽子）。
- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 小黑猫只是蹲在角落看，没有参与核心动作。

## 迭代方法

- 太普通：让小黑猫成为动作主体，用爪子或尾巴做核心动作。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 absurd、deadpan、not overly cute。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 文字错：重生成并减少标注数量。
- 猫不对：检查提示词里的 IP 描述是否准确（黄色眼睛、粉色鼻子、腮红、团子身）。

## 交付判断

高质量图应该让读者先觉得"有点怪"，然后 1 秒内看懂结构，同时注意到"这只猫在干活"。
